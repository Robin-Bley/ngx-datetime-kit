# i18n Guide

## Locale-aware Rendering

The library uses `LOCALE_ID` from Angular for:
- Weekday names (Monday, Montag, Lundi, ...)
- Month names (January, Januar, Janvier, ...)
- First day of week (Sunday vs Monday depending on locale)

### Register Locale Data

Angular ships only `en-US` data by default. For other locales, import the data:

```typescript
// main.ts or app.config.ts
import { registerLocaleData } from '@angular/common';
import localeDe from '@angular/common/locales/de';

registerLocaleData(localeDe);
```

Or use the global variant:

```typescript
import '@angular/common/locales/global/de';
```

Then set `LOCALE_ID`:

```typescript
providers: [
  { provide: LOCALE_ID, useValue: 'de-DE' },
  provideNgxDatetimeKit(),
]
```

## Configuring Labels

All UI text is injectable via `NGX_LABELS`. Override globally or per-component:

```typescript
// Global (in app.config.ts)
provideNgxDatetimeKit({
  labels: {
    apply: 'Übernehmen',
    cancel: 'Abbrechen',
    reset: 'Zurücksetzen',
    today: 'Heute',
    last7Days: 'Letzte 7 Tage',
    // ... see NgxLabels interface for all keys
  }
})

// Per-component
@Component({
  providers: [
    { provide: NGX_LABELS, useValue: { apply: 'Confirm', cancel: 'Dismiss' } }
  ]
})
```

## Per-Component Locale Override

Each component can use a different locale by overriding the adapter:

```typescript
@Injectable()
class FrenchAdapter extends NgxNativeDateTimeAdapter {
  constructor() {
    super();
    this.setLocale('fr-FR');
  }
}

@Component({
  providers: [{ provide: NGX_DATE_TIME_ADAPTER, useClass: FrenchAdapter }],
})
export class FrenchPickerComponent {}
```

## RTL Support

RTL layout is handled via CSS `direction: rtl` on a parent element. The components use logical CSS properties where possible. Explicit RTL testing is recommended for Arabic/Hebrew locales.
